this本质

this是在运行时绑定，并不是在编写时绑定，它的上下文取决于函数调用时的各种条件。

当一个函数被调用时，会创建一个活动记录（有时候也称为执行上下文）。这个记录会包含函数在哪里调用（调用栈）、函数的调用方式、传入的参数等信息。this是这个记录的一个属性，会在函数执行的过程中用到，这是this的本质。

this绑定规则

  1. 函数是否在new中调用（new绑定）？如果是的话this绑定的是新创建的对象。
  2. 函数是否通过call、apply（显式绑定）或者硬绑定调用？如果是的话，this绑定到指定对象。
  3. 函数是否在某个上下文中调用（隐式绑定）？如果是的话，this绑定的是那个上下文对象。
  4. 如果都不是的话，使用默认绑定。严格模式下绑定到undefined，否则绑定到全局对象。

在此复习一下new调用函数时执行的操作：

  1. 创建一个新对象
  2. 执行[[prototype]]连接
  3. this绑定到新对象
  4. 如果函数没有返回其他对象，new表达式中的函数调用并返回这个新对象

this绑定时的例外

  • 被忽略的this

    如果把null或者undefined当作this的绑定对象传入call、apply或者bind，这些值会在调用时被忽略，实际应用的是默认绑定规则（使用bind进行柯里化时可能会这么用）。

    这样的风险是如果函数真的用到了this，会改变全局对象，更安全的做法是使用一个空的非委托对象（DMZ）作为this的绑定对象传入。
